## Über

Das ArjunaCRM GitHub-Repository beherbergt eine Vielzahl von Ressourcen wie Quellcode, Dokumentation, Diskussionen und Problemverfolgung. Hier können Sie auf den vollständigen Quellcode hinter ArjunaCRM zugreifen.

[Besuchen Sie ArjunaCRM auf GitHub](https://github.com/vedpragyabharat/arjuna)

## Mitwirken

Zur Mitarbeit am ArjunaCRM Projekt auf GitHub beizutragen ist eine lohnende Möglichkeit, die Software zu verbessern, die Sie nutzen. Ob Sie Fehler beheben, Funktionen vorschlagen oder die Dokumentation verbessern, Ihre Beiträge sind willkommen.

### Probleme melden

Tritt ein Problem auf? [Ein Problem erstellen](https://github.com/vedpragyabharat/arjuna/issues/new) auf GitHub und so viele Details wie möglich angeben.

<img src="/images/user-guide/github/new-issue.png" style={{width:'100%'}}/>

### Funktionen vorschlagen

Welche Verbesserungen möchten Sie bei ArjunaCRM sehen? Egal welches technische Wissen Sie haben, Sie können [hier an der Diskussion teilnehmen](https://github.com/vedpragyabharat/arjuna/discussions).

<img src="/images/user-guide/github/github-conversations.png" style={{width:'100%'}}/>

### Eine Funktion programmieren

Beginnen Sie Ihre Reise, indem Sie Aufgaben für Anfänger finden:

1. Navigieren Sie zur **[Issues](https://github.com/vedpragyabharat/arjuna/issues)**-Registerkarte im ArjunaCRM-Repository.
2. Filtern Sie nach dem **[Good First Issue](https://github.com/vedpragyabharat/arjuna/labels/good%20first%20issue)**-Label, um Aufgaben für Neulinge zu finden.
3. Wählen Sie ein Problem aus, forken Sie das Repository und beginnen Sie mit dem Beitrag.

<img src="/images/user-guide/github/good-first-issues.png" style={{width:'100%'}}/>

Stellen Sie sicher, dass Sie dem Problem zugewiesen sind, um überschneidende Arbeiten mit anderen Mitwirkenden zu vermeiden.

### Verhaltensregeln

Denken Sie daran, während des gesamten Beitragsprozesses die [Verhaltensregeln](https://github.com/vedpragyabharat/arjuna/blob/main/.github/CODE_OF_CONDUCT.md) von ArjunaCRM einzuhalten.
